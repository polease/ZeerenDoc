# Ongoing Project / Work 继续协作流程

## 场景概述
- **项目周期长、步骤多**：在持续演进过程中，Human Worker 可能离开后又重新回来。  
- **目标**：让回归的成员快速了解进度，并在需要决策或输入时立即介入。

## 1. 进入既有 Project
1. 从列表选择 **进行中的 Project / Work**。  
2. 系统加载该 Project 的完整上下文。

## 2. 快速全貌呈现
| 信息块 | 内容 |
|--------|------|
| **已完成步骤** | 展示已结束的任务及其状态 |
| **产生的 Deliverables** | Resource 列表中已产出的文档、代码、资产等 |
| **当前进行中** | 高亮当前正在执行的 Task / Step |
| **待决策或待输入** | 明确指出需要 Human Worker 介入的节点 |

## 3. Human ↔ Work Engine 交互
- **Conversation 面板**：集中显示任务讨论、决策记录与所需输入。  
- **控制指令**：Human Worker 可对单个 Task 或引擎整体执行 _Pause / Resume / Update_ 操作。  
- **异步通知**：离线时仍可接收需要决策的提醒，返回后继续对话。  

## 4. Worker Continuation 机制
1. **多次进入**：Human Worker 可在项目生命周期内多次离开与回归。  
2. **状态保持**：每次返回自动定位到 _待办 Task_。  
3. **迭代循环**：项目完成后仍可重新打开，继续优化或迭代交付物。  

> 该机制保证了长周期项目在成员流动情况下依旧 **可追踪、可接续、可迭代**，直至项目终结甚至后续增量优化。
