# New Work Journey

## 1. 获取用户意图
- 用户进入系统并说明 **“我想做什么”**（如撰写技术提议、调研报告、开发应用等）。
- 系统判定该意图为 **新意图**，而非现有项目的后续沟通。

## 2. 意图解析与 Work Domain 匹配
1. 系统解析用户意图 → 找到最匹配的 **Work Domain**。  
2. 进入对应 **Playbook**，根据规则自动生成 **WBS**（Work Breakdown Structure）。

## 3. WBS 生成与展示
- 系统根据 Playbook 拆解任务，生成含 **任务、状态** 等字段的 WBS 表格。  
- 用户可在前端实时查看并跟踪进度。

## 4. 任务下发与执行
| 任务类型 | 执行方式 |
|---------|----------|
| **Agent 任务** | 调用大模型自动完成并回写资源 |
| **Human 任务** | 通过 WebSocket 分配给具体人员；上传文字/图片或直接编辑资源 |

## 5. 实时状态与控制
- 系统持续更新任务状态，用户可：  
  - 查看当前 **Agent / Human** 执行情形  
  - 发送 **暂停 / 停止** 指令  
  - 对需决策步骤触发 **Ask for Human Review** 并暂停等待人工处理  

## 6. 可视化协作 Canvas
- 前端提供 **无限画布 (Canvas)**：  
  - 每当任务触及资源，自动在画布中打开  
  - 用户实时看到编辑操作

## 7. 交付与预览
- 任务结束后：  
  - 系统生成 **Summary 报告**  
  - 若为应用项目，可触发构建并提供 **在线预览**

## 8. 成本与审计
- 全程记录：任务用时、成本、Token 用量等  
- 在合适位置向用户展示，便于后续审计与优化
